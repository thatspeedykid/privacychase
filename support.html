Frequently Asked Questions

Both Apps I lost my data. Can you recover it?

Unfortunately, no. Because all data is stored only on your device and never sent to any server, we have no copy of it. This is by design — it protects your privacy. Going forward, use the Copy Backup feature regularly. It encodes your entire dataset into a single line of text you can paste anywhere (Notes, email, a text file). One tap, instant restore on any device.

Both Apps How do I move my data to a new device?

Tap Copy Backup on your old device. Paste the text somewhere you can access from your new device (Notes, iMessage, email). On the new device, open the app, tap Paste & Restore, paste the text, and you're done. Everything comes back instantly.

Flowe Windows shows a SmartScreen warning. Is this safe?

Yes. Flowe is currently unsigned — code signing certificates cost $200–400/year for Windows. Click More info → Run anyway to proceed. The source code is fully public on GitHub if you'd like to verify what's in the installer. You're also welcome to build from source yourself.

Flowe How do I install on iOS without the App Store?

Until the App Store release, iOS requires sideloading via Sideloadly. Download the .ipa from the GitHub Releases page and follow Sideloadly's instructions. A free Apple ID is sufficient.

Flowe I upgraded and my data is gone on Android.

If you had an older version of Flowe (package ID com.example.flowe), the new version (com.privacychase.flowe) auto-migrates your data on first launch. If migration didn't work, you should have used Copy Backup before upgrading — see the README for migration details. Email us and we'll try to help.

Fuelle Food searches aren't returning results.

Fuelle uses the USDA FoodData Central API for food searches. Check your internet connection — this is the only part of Fuelle that requires network access. If it's still not working, the USDA API may be temporarily unavailable. Try again later or report the issue on GitHub.

Both Apps Will there be a cloud sync feature?

Not in the traditional sense. Cloud sync requires a server, accounts, and trust — all of which contradict the core philosophy. The backup/restore feature is designed to cover most portability needs without any of that. If there's strong demand for something like optional encrypted sync, it would be considered carefully with privacy as the top constraint.

Both Apps I found a bug. How do I report it?

The best way is to open a GitHub Issue in the relevant repository (links above). Include your platform (Windows / macOS / Linux / Android / iOS), the app version, and a description of what happened. If it's sensitive, email us instead.

Both Apps Can I request a feature?

Yes — open a GitHub Issue and tag it as a feature request. No promises on timeline, but all suggestions are read. This is a one-person operation, so realistic expectations are appreciated.
